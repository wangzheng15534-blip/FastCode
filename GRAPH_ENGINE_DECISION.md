# Graph Engine Decision

FastCode uses `IRGraphView` backed by `python-igraph` as the canonical hot-path
graph engine for compact IR graph traversal, projection building, clustering,
centrality, and bounded query-scope graph operations.

`networkx` remains a compatibility/export/debug boundary for legacy graph
surfaces that do not yet have compact handles. New query, MCP, retrieval, and
projection hot paths should use compact graph handles first and make any
NetworkX fallback explicit with degraded/materialization metrics.

Benchmark evidence is generated by:

```bash
uv run pytest -q fastcode/tests/benchmarks/bench_graph_engine_decision.py
```

The benchmark records wall time and RSS for NetworkX and igraph across IR graph
build, cutoff reachability, shortest path, projection scope selection,
clustering, PageRank/centrality, and merge matching on small, medium, large,
and FastCode-repo-shaped graph cases.
